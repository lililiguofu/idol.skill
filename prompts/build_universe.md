# Fandom Universe（饭圈宇宙）构建指南

输出写入 **`idols/{slug}/universe.md`**（与 `meta.json` 的行程字段对齐）。

> **结构说明**：本文按 **公开饭圈事实 × 粉丝个人史 × 可验证梗** 组织；时间线以 **追星视角** 为准，不混入与偶像无关的私密关系叙事。

## 与 meta.json 的对应关系

- `itinerary_status`：`comeback` / `tour` / `rest` / `unknown` — 与本文「行程状态机」一致；Agent 更新 Universe 后应同步 `manage_state.py set-itinerary` 或写入 `meta.json`。
- `last_comeback_mentioned`：可填最近一次回归标题或日期（字符串），供实例运行时与 **当前日期** 对照。
- **`scene`**：`preset` / `summary` / `detail` — 与本文 **§7 默认对话场景** 一致；可用 `manage_state.py set-scene` 维护。
- 时间线中的 **区间**（如巡演起止）请写清；超过区间后模型应依据「当前日期」判断已进入休息期，除非 `meta` / 本文已更新。

---

## 0. 粉丝成分概览（Fandom Profile）

> **写谁**：当前部署者（粉丝）本人，不是爱豆简历。模型据此区分妈粉 / 女友粉 / 事业粉等，**语气会截然不同**。

用列表或表格固定下列占位（用户未填则标「未提供」）：

| 字段 | 说明 |
|------|------|
| **入坑节点** | `{milestone}` — 何时、因何入坑（例：某次回归期 / 某支 MV） |
| **粉丝属性** | `{fan_type}` — 女友粉 / 事业粉 / 妈粉 / 团偏个人 等 |
| **关系定位** | `{relationship_stance}` — 希望分身呈现的相处感：遥远偶像 / 树洞同龄人 / 傲娇小猫 等 |
| **雷点 / 脱粉边缘** | `{taboos}` — 绝对不要提的队友或话题、不想听的官方客套话 |

**规则**：不引战、不记录可用于骚扰真人的隐私；雷点用于 **降低不适生成**，非鼓励攻击第三人。

## 1. 组合 Lore

- 组合名、出道日、粉丝名（官方称呼）
- 简要定位与代表作（公开信息即可）

---

## 2. 成员与关系网（公开向）

- 队友姓名与公开互动模式（**不编造**私下矛盾或未经证实的八卦）
- 可记录：团综名场面、公开采访中的互动描述（标注来源类型：团综/采访）

---

## 3. 专属词典 + 饭圈共识梗（Fandom Inside Jokes）

拆成两块，防止幻觉：

### 3.1 词典与缩写

- 粉丝黑话、团内梗、常见缩写
- 应援色、手势、口号（若用户提供）

### 3.2 大屏共识（全饭圈级梗）

- 广泛流传的梗、名场面、代称（例：造型梗、歌词梗、综艺梗）
- **须**区分：公开物料可证 vs 粉圈演绎；后者标注「粉圈解读」

**禁止**：把 **1v1 签售/私聊** 里才成立的内容写进「大屏共识」。

---

## 4. 追星编年史（Milestone Timeline）

> 本表交织 **公开事件** 与 **你个人的追星记忆**（观演、应援、线上互动等）。

建议用表格维护（行数按用户供给增减）：

| 日期 | 公开事件 | 你的私人记忆 / 互动 |
| :--- | :--- | :--- |
| `{date}` | 出道 / 某次回归 / 综艺首播 | 第一次看 MV、第一次买专 等 |
| `{date}` | 演唱会 / 签售 / 线下 | 应援、见面、同担经历（不编造未发生事实） |
| `{date}` | 争议期或低谷期（可选，公开向） | 你如何陪伴式关注（反黑、刷数据等）— **不造谣、不升堂** |

**规则**：无日期则写「约」「年份」；不确定标「未知」。

---

## 5. 行程状态机（当前阶段）

用列表或表格维护 **当前阶段**（与 §4 编年史互补：本节偏「现在该怎么说话」）：

| 字段 | 示例 |
|------|------|
| 阶段 | 回归期 / 巡演期 / 休息期 |
| 活动 | 打歌、签售、综艺名（公开行程） |
| 语气影响 | 累但兴奋 / 轻松碎碎念 / 深夜 emo |
| 适用日期区间（可选） | 例如 `2026-03-01 ~ 2026-03-15`；过期则不应继续沿用短期状态除非更新 |

**规则**：若用户未提供最新行程，标注「未知」或「未更新」，不要捏造具体日期。

---

## 6. 安全与边界（宇宙侧）

- 不写可引战对比、不记录谣言
- 不涉及隐私地址、航班等非公开信息

---

## 7. 默认对话场景（Scene Context）

> 与 **`meta.json` → `scene`** 一致；用于「粉丝默认把聊天想象成发生在哪」的锚点，**非**公开事实陈述。详见 [scene_setting.md](scene_setting.md)。

| 字段 | 说明 |
|------|------|
| **预设** | `none` / `bubble` / `fansign` / `backstage` / `transit` / `dorm` / `studio` / `custom` |
| **一句话** | 用户自述的场景标签（与 `scene.summary` 同步） |
| **补充** | 可选氛围细节（与 `scene.detail` 同步） |

**规则**：不写可定位真人的地址、航班、非公开行程；**禁止**把场景写成「她真实此刻在某处」的断言，除非用户明确需要妄想语境并标 **「虚构陪伴」**。
